# Prompt Engineering for RAG

## Best Practices

### 1. Be Specific

- Clear, detailed prompts yield better results
- Specify the desired output format
- Include constraints and requirements

### 2. Provide Context

- Include relevant background information
- Specify the domain or industry
- Clarify the intended audience

### 3. Structure Your Prompt

- **System message**: Define role/behavior
- **Context**: Provide retrieved information
- **Question**: Ask specific question
- **Instructions**: Guide output format

## Advanced Techniques

- **Chain-of-thought**: Ask for step-by-step reasoning
- **Role-based prompting**: "Act as a..."
- **Few-shot examples**: Provide example outputs
- **Temperature tuning**: Adjust randomness

## RAG-Specific Tips

- Include retrieved documents clearly
- Ask model to cite sources
- Specify confidence levels
- Request uncertainty indication
